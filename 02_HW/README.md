# Требования к программам

## Общие требования

1. Задачи оцениваются независимо в двух группах: задачи 1–5 и задачи 6–9.
2. Программа должна получать все параметры в качестве аргументов командной строки.

## Аргументы командной строки

### Для задачи 1

1. `fin` — имя входного файла
2. `fout` — имя выходного файла
3. `s` — строка `s`
4. `t` — строка `t`
5. `m` — параметр `m` алгоритма поиска

Пример запуска:

```bash
./a01.out a.txt b.txt "abcd" "(){}[] ;+-*/=" 2
```

Это означает:
- читать строки из файла `a.txt`
- выводить результат в файл `b.txt`
- параметр `s = "abcd"`
- параметр `t = "(){}[] ;+-*/="`
- параметр `m = 2`

## Общий результат работы функций

- Результатом работы функции является **измененный файл**
- В `main` выводится **возвращаемое значение функции**

Формат вывода в `main`:

```cpp
printf ("%s : Task = %d Result = %d Elapsed = %.2f\n",
        argv[0], task, res, t);
```

Где:

- `argv[0]` — имя программы
- `task` — номер задачи
- `res` — результат работы функции
- `t` — время работы функции

Формат должен соблюдаться строго.

# Задачи

## Задача 1

Написать функцию, которая:

**Аргументы**

- имя входного файла `a`
- имя выходного файла `b`
- строки `s`, `t`
- число `m`
- указатель на целое `r`

**Функция должна**

Выводить в файл `b` строки файла `a`, которые имеют **общее слово со строкой `s`**.

**Определения**

- слово — последовательность символов без пробелов
- пробельный символ — любой символ из строки `t`

**Алгоритм**

1. Слова строки `s` заранее помещаются в **B‑дерево строк** с основанием `m`
2. Строки и массивы в узлах дерева размещаются через:

```
std::unique_ptr
```

3. Каждая строка файла:
   - разбивается на слова
   - каждое слово ищется в B‑дереве

**Результат**

- в `r` записывается количество выведенных строк
- возвращается `io_status`

---

## Задача 2

Написать функцию, которая выводит в файл `b` строки файла `a`, **состоящие только из слов строки `s`**.

**Используется**

- **красно‑черное дерево строк**

**Особенности**

- строки хранятся как

```
std::unique_ptr<char[]>
```

Алгоритм:

1. слова строки `s` заносятся в красно‑черное дерево
2. каждая строка файла разбивается на слова
3. слова ищутся в дереве

Результат — количество выведенных строк.

---

## Задача 3

Функция заменяет слова из строки `s` на слова из строки `x`.

**Соответствие**

- по номеру слова

Если:

- слов в `s` больше чем в `x` → недостающие считаются пустыми
- слов в `x` больше → лишние игнорируются

**Используется**

- **AVL‑дерево пар строк**

```
(слово из s → слово из x)
```

Алгоритм:

1. пары слов `(s[i], x[i])` помещаются в AVL‑дерево
2. строка файла разбивается на слова
3. каждое слово ищется в дереве
4. если найдено — заменяется

---

## Задача 4

Вывести строки файла `a`, в которых есть слово, удовлетворяющее условию из строки `x` относительно слова из `s`.

**Допустимые условия**

```
<
>
<=
>=
=
<>
```

Количество условий = количеству слов в `s`.

Если условий несколько — используется **логическое ИЛИ**.

Пример:

```
s = "ab cd"
x = "= >"
```

Подходят слова:

- равные `"ab"`
- или лексикографически больше `"cd"`

---

## Задача 5

Аналогично задаче 4, но применяется **логическое И**.

Все слова строки должны удовлетворять условиям.

Пример:

```
s = "ab cd"
x = "> <"
```

Подходят слова:

- больше `"ab"`
- и меньше `"cd"`

соответствует **любому одному символу**

Экранирование:

```
%
```

Значение:

```
% → 0 или более символов
```

Допустимые формы:

```
text%
%text
%text%
text1%text2
```

Экранирование:

```
\%
\\
```

## Задача 8

Шаблон вида:

```
[n-m]
```

Соответствует **любому символу из диапазона `n..m`**.

Экранирование:

```
\[
\]
\\
```

Ошибки:

- одиночный `\` в конце
- одиночный `[` или `]`

Шаблон вида:

Соответствует **любому символу вне диапазона `n..m`**.

Экранирование:

```
\[
\]
\b
\\
```

Ошибки:

- одиночный `\`
- несбалансированные `[` `]`
